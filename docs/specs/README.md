# Cross-cutting specs

Specs for initiatives that **span multiple domains** (frontend + backend + infra) live here,
one folder per initiative: `docs/specs/<initiative>/{requirements,design,test-cases,tasks}.md`.

Feature-scoped specs live next to their feature instead, e.g.
`frontend/src/features/<feature>/docs/`.

Copy [`_template/`](./_template/) to start a new spec. Process:
[`../development-process.md`](../development-process.md).
