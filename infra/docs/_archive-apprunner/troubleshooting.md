## ❌ Common Issues

### 1. Docker Build Fails

**Symptoms**: Error during `docker build` or CDK deployment

**Solutions**:

```bash
# Test build locally
cd frontend
npm run build

# Check for syntax errors
npm run lint

# Test Docker build
docker build -t test .

# Check Docker is running
docker ps

# Clear Docker cache
docker system prune -a
```

**Common Causes**:
- Syntax errors in code
- Missing dependencies
- Docker not running
- Insufficient disk space

### 2. CDK Bootstrap Error

**Symptoms**: `This stack uses assets, so the toolkit stack must be deployed`

**Solution**:

```bash
cd infra
npx cdk bootstrap aws://ACCOUNT-ID/REGION

# Or use default account/region
npx cdk bootstrap
```

### 3. Permission Denied Errors

**Symptoms**: `AccessDenied` or `UnauthorizedOperation`

**Solutions**:

```bash
# Verify credentials
aws sts get-caller-identity

# Check IAM permissions needed:
# - ECR (push images)
# - AppRunner (create service)
# - IAM (create roles)
# - CloudWatch (create alarms)
# - SNS (create topics)

# Reconfigure AWS CLI
aws configure
```

### 4. Service Won't Start

**Symptoms**: Service status is `CREATE_FAILED` or `OPERATION_FAILED`

**Solutions**:

```bash
# Check CloudWatch logs
aws logs tail /aws/apprunner/nextjs-admin-dashboard --follow

# Common issues:
# - Port mismatch (should be 3000)
# - Environment variables incorrect
# - Health check failing
# - Container crashes on startup

# Test locally
cd frontend
docker build -t test .
docker run -p 3000:3000 test
# Visit http://localhost:3000
```

### 5. Health Check Failing

**Symptoms**: Service keeps restarting, health check errors in logs

**Solutions**:

```bash
# Verify health check endpoint
curl http://localhost:3000/

# Check health check config in app-stack.ts:
# - path: '/'
# - port: '3000'
# - protocol: 'HTTP'

# Test Docker container health
docker run -p 3000:3000 test
curl http://localhost:3000/
```

### 6. High Memory/CPU Usage

**Symptoms**: Service slow, alarms triggering

**Solutions**:

```typescript
// Increase instance size in config/app-config.ts
instanceConfig: {
  cpu: '2 vCPU',    // Increase from 1
  memory: '4 GB',   // Increase from 2
}
```

```bash
# Redeploy
npm run deploy:dev
```

### 7. Custom Domain Not Working

**Symptoms**: Domain doesn't resolve to service

**Solutions**:

1. **Verify CNAME records**:
   - Go to AWS Console → App Runner → Service → Custom domains
   - Copy the validation CNAME records
   - Add to your DNS provider

2. **Wait for propagation**:
   - DNS changes can take 5 minutes to 48 hours
   - Check with: `dig admin.example.com`

3. **Verify domain in config**:
   ```typescript
   customDomain: 'admin.example.com'  // No https://
   ```

### 8. Environment Variables Not Applied

**Symptoms**: App can't access environment variables

**Solutions**:

```typescript
// Check config/app-config.ts
environmentVariables: {
  DATABASE_URL: 'value',  // Make sure it's here
}

// Redeploy
npm run deploy:dev

// Verify in AWS Console
// AppRunner → Service → Configuration → Environment variables
```

### 9. Deployment Takes Too Long

**Symptoms**: Deployment stuck or very slow

**Solutions**:

```bash
# Check deployment status
aws apprunner list-operations --service-arn <arn>

# Common causes:
# - Large Docker image (optimize Dockerfile)
# - Slow internet connection
# - AWS service issues

# Optimize image size:
# - Use .dockerignore
# - Multi-stage builds (already configured)
# - Remove unnecessary dependencies
```

### 10. "Stack Already Exists" Error

**Symptoms**: CDK deploy fails with stack exists error

**Solutions**:

```bash
# Check existing stacks
aws cloudformation list-stacks

# Update existing stack
npm run deploy:dev

# Or delete and recreate
npm run destroy:dev
npm run deploy:dev
```

## 💡 Best Practices

1. **Always test locally first**:
   ```bash
   docker build -t test . && docker run -p 3000:3000 test
   ```

2. **Use CDK diff before deploying**:
   ```bash
   npm run diff:dev
   ```

3. **Monitor logs during deployment**:
   ```bash
   aws logs tail /aws/apprunner/nextjs-admin-dashboard --follow
   ```

4. **Keep dev and prod separate**:
   - Use different AWS accounts or regions
   - Different configuration files
   - Different domain names

5. **Regular cleanup**:
   - Delete unused ECR images
   - Remove old CloudWatch logs
   - Delete dev stacks when not in use

  --filter-pattern "ERROR"

# Filter for specific time range

  --log-group-name /aws/apprunner/nextjs-admin-dashboard \
