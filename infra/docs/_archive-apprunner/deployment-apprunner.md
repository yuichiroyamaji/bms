# Deployment Guide

> 🤖 This documentation was generated by Kiro AI

## Prerequisites

1. **AWS CLI** installed and configured:
   ```bash
   aws configure
   ```

2. **Node.js** 18+ installed

3. **Docker** installed and running (for building container images)

## Initial Setup

### 1. Bootstrap CDK (First Time Only)

```bash
cd infra
npm install
npx cdk bootstrap
```

This creates the necessary AWS resources for CDK deployments.

### 2. Configure Your Application

Edit `infra/config/app-config.ts` to set:
- Your email for CloudWatch alarms
- Custom domain (if you have one)
- Environment variables
- Instance size (CPU/Memory)

## Deployment Commands

### Deploy to Development

## Custom Domain Setup

If you want to use a custom domain:

1. **Update configuration** in `infra/config/app-config.ts`:
   ```typescript
   customDomain: 'admin.example.com'
   ```

2. **Deploy the stack**:
   ```bash
   npx cdk deploy AppStack-dev -c environment=dev
   ```

3. **Get DNS validation records**:
   - Go to AWS Console → App Runner → Your Service → Custom domains
   - Copy the CNAME records

4. **Add CNAME records to your DNS provider**:
   - Add the validation records to your domain's DNS settings
   - Wait for validation (can take a few minutes to hours)

## Environment Variables

### Adding Environment Variables

Edit `infra/config/app-config.ts`:

```typescript
environmentVariables: {
  DATABASE_URL: 'postgresql://...',
  API_KEY: 'your-api-key',
  NEXT_PUBLIC_API_URL: 'https://api.example.com',
}
```

### Using AWS Secrets Manager (Recommended for Sensitive Data)

For sensitive data like database passwords:

1. Create a secret in AWS Secrets Manager
2. Grant the AppRunner instance role access to the secret
3. Reference it in your application code

## Monitoring & Alarms

### CloudWatch Alarms

The stack automatically creates alarms for:
- **CPU Utilization** > 80%
- **Memory Utilization** > 80%
- **4xx Errors** > 50 in 5 minutes

You'll receive email notifications when alarms trigger.

### Viewing Logs

```bash
# Get service name
aws apprunner list-services

# View logs in CloudWatch
aws logs tail /aws/apprunner/nextjs-admin-dashboard --follow
```

Or use the AWS Console:
- CloudWatch → Log groups → `/aws/apprunner/nextjs-admin-dashboard`

## Scaling Configuration

### Adjusting Instance Size

Edit `infra/config/app-config.ts`:

```typescript
instanceConfig: {
  cpu: '2 vCPU',    // Options: 0.25, 0.5, 1, 2, 4 vCPU
  memory: '4 GB',   // Options: 0.5-12 GB
}
```

### Auto Scaling (Future Enhancement)

AppRunner supports auto-scaling. To enable:
- Add `autoScalingConfigurationArn` to the service configuration
- Configure min/max instances based on traffic

## Cost Optimization

### Current Configuration Cost
- **1 vCPU + 2 GB RAM**: ~$3-5 USD/month (minimal traffic)
- Scales with usage

### Tips to Reduce Costs
1. Use smaller instance sizes for dev environments
2. Delete dev stacks when not in use:
   ```bash
   npx cdk destroy AppStack-dev -c environment=dev
   ```
3. Monitor CloudWatch metrics to right-size instances

## Troubleshooting

### Build Fails

```bash
# Test Docker build locally
cd frontend
docker build -t nextjs-test .
docker run -p 3000:3000 nextjs-test
```

### Deployment Fails

```bash
# Check CDK logs
npx cdk deploy AppStack-dev -c environment=dev --verbose

# Verify AWS credentials
aws sts get-caller-identity
```

### Service Not Starting

1. Check AppRunner logs in CloudWatch
2. Verify environment variables are correct
3. Check health check endpoint is responding

### Custom Domain Not Working

1. Verify CNAME records are added correctly
2. Wait for DNS propagation (can take up to 48 hours)
3. Check domain validation status in AppRunner console
