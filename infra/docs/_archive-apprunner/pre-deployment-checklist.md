# Pre-Deployment Checklist

> 🤖 This documentation was generated by Kiro AI

Use this checklist before deploying to ensure everything is configured correctly.

## ✅ Prerequisites

- [ ] AWS CLI installed and configured (`aws --version`)
- [ ] AWS credentials configured (`aws sts get-caller-identity`)
- [ ] Node.js 18+ installed (`node --version`)
- [ ] Docker installed and running (`docker --version`)
- [ ] CDK bootstrapped (`npx cdk bootstrap`)

## ✅ Configuration

- [ ] Updated email in `config/app-config.ts`
- [ ] Reviewed environment variables in `config/app-config.ts`
- [ ] Set appropriate instance size (CPU/Memory)
- [ ] Configured custom domain (if needed)
- [ ] Reviewed both `devConfig` and `prodConfig`

## ✅ Testing

- [ ] Frontend builds successfully (`cd frontend && npm run build`)
- [ ] Docker image builds locally (`cd frontend && docker build -t test .`)
- [ ] Docker container runs locally (`docker run -p 3000:3000 test`)
- [ ] Application works at http://localhost:3000
- [ ] Infrastructure compiles (`cd infra && npm run build`)
- [ ] CDK synth works (`npx cdk synth AppStack-dev -c environment=dev`)

## ✅ Security

- [ ] No sensitive data in code (use Secrets Manager)
- [ ] Environment variables reviewed
- [ ] IAM roles have minimal permissions
- [ ] Health check endpoint is public (required for AppRunner)

## ✅ Cost Review

- [ ] Reviewed instance size for environment
- [ ] Understand pricing: ~$3-5/month for 1 vCPU + 2 GB
- [ ] Plan to delete dev stacks when not in use
- [ ] CloudWatch alarm email configured

## ✅ Deployment Plan

- [ ] Deploying to correct environment (dev/prod)
- [ ] Reviewed changes with `npm run diff:dev` or `npm run diff:prod`
- [ ] Team notified (for production deployments)
- [ ] Deployment window scheduled (for production)
- [ ] Rollback plan prepared

## ✅ Post-Deployment

- [ ] Service URL accessible
- [ ] Application loads correctly
- [ ] CloudWatch logs showing activity
- [ ] No errors in logs
- [ ] Health checks passing
- [ ] Alarm email received (confirm subscription)
- [ ] Custom domain working (if configured)
- [ ] Environment variables applied correctly

## 🚀 Ready to Deploy?

If all items are checked, proceed with deployment:

```bash
# Development
npm run deploy:dev

# Production
npm run deploy:prod
```

## 🆘 If Something Goes Wrong

1. **Check CloudWatch Logs**:
   ```bash
   aws logs tail /aws/apprunner/nextjs-admin-dashboard --follow
   ```

2. **Review Service Status**:
   ```bash
   aws apprunner list-services
   ```

3. **Rollback** (if needed):
   ```bash
   npm run destroy:dev  # or destroy:prod
   ```
   Then redeploy previous version

4. **Get Help**:
   - Review [DEPLOYMENT.md](./DEPLOYMENT.md)
   - Check [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)
   - Check AWS AppRunner console
   - Review CDK error messages

## 📝 Notes

- First deployment takes 5-10 minutes
- Subsequent deployments are faster (3-5 minutes)
- Zero-downtime rolling deployments
- Old container images are kept in ECR (can be cleaned up manually)
